# Implementação: Gerenciamento de Voluntários (v1.2.0)

Este documento registra os detalhes técnicos e as funcionalidades integradas para permitir que os administradores gerenciem os voluntários da aplicação.

## 1. Problema
O botão de "Gerenciar Voluntários" no painel administrativo estava sem comportamento definido (`/* TODO */`) e a rota correspondente (`admin_volunteers`) não possuía uma tela implementada ou vinculada no Navigation Host da `MainActivity`.

## 2. Solução
Implementou-se a tela de controle total sobre a base de voluntários cadastrados, com interface premium otimizada para o tema escuro ("Sophisticated Dark").

### Funcionalidades integradas:
- **Barra de Pesquisa Dinâmica:** Filtragem em tempo real de voluntários por Nome, E-mail ou Telefone.
- **Alteração de Nível Técnico:** Menu suspenso (DropdownMenu) para atualizar instantaneamente o nível de competência técnica do operador (`Iniciante`, `Intermediário`, `Avançado` ou `Líder`).
- **Alteração de Perfil de Acesso:** DropdownMenu para alternar a função do usuário entre `VOLUNTEER` (voluntário) e `ADMIN` (administrador).
- **Ativação e Inativação (Status):** Interruptor interativo (`Switch`) estilizado para alternar o status do voluntário entre `ACTIVE` (verde) e `INACTIVE` (vermelho).
- **Dados de Contato Rápidos:** Exibição elegante do e-mail e telefone de cada usuário.
- **Navegação Fluida:** Botão de retrocesso na barra superior que retorna com precisão ao painel de controle principal.

## 3. Arquivos Criados ou Modificados
- `/app/src/main/java/com/example/ui/screens/AdminVolunteersScreen.kt`: Arquivo criado com a tela de gerenciamento completa e o componente `VolunteerCard`.
- `/app/src/main/java/com/example/viewmodel/HomeViewModel.kt`: Adicionadas as funções de persistência no repositório (`toggleUserStatus`, `updateUserLevel` e `updateUserRole`).
- `/app/src/main/java/com/example/ui/screens/AdminDashboardScreen.kt`: Adicionado o parâmetro `onNavigateToVolunteers` e configurado no clique do botão.
- `/app/src/main/java/com/example/MainActivity.kt`: Configurada a rota `Route.AdminVolunteers.route` vinculando-a ao novo composável `AdminVolunteersScreen`.

## 4. Testes e Validação
- O código compilou perfeitamente via `compile_applet`.
- O comportamento do switch de estado e dropdowns foi validado integrando as alterações reativas diretamente ao StateFlow `allUsers` do ViewModel.
