# Implementação: Modo Desenvolvedor / Bypass Administrativo (v1.1.0)

Este documento registra os detalhes técnicos e a motivação para a implementação do Modo Desenvolvedor com botão de bypass administrativo.

## 1. Problema
Durante a fase de desenvolvimento e testes de usabilidade, digitar repetidamente as credenciais do usuário Administrador (`admin@admin.com` / `admin`) desacelera o ciclo de feedback e validação do fluxo completo de telas do painel.

## 2. Solução
Foi criada uma variável de ambiente `DESENVOLVIMENTO=true` no arquivo `.env` (exposta via `.env.example`).
O aplicativo lê essa variável através do `com.example.BuildConfig.DESENVOLVIMENTO`.
- Se a variável estiver definida como `"true"`, a tela de login exibe um botão de acesso rápido: **"Acessar como Admin (Modo Dev)"**.
- Ao clicar no botão, o app simula as credenciais e executa o login do administrador padrão de forma instantânea.

## 3. Arquivos Modificados
- `/.env.example`: Adicionada a declaração da variável de ambiente com comentário explicativo.
- `/.env`: Arquivo de ambiente local gerado com a variável ativa por padrão em desenvolvimento.
- `/app/src/main/java/com/example/ui/screens/AuthScreens.kt`: Importação de `BuildConfig` e implementação do condicional de renderização do botão de acesso rápido com layout seguindo a estética "Sophisticated Dark".

## 4. Testes e Validação
- Executado o comando `compile_applet` obtendo sucesso na compilação.
- Verificada a correta injeção e tipagem através do plugin de segredos do Gradle.
